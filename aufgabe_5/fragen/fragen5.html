Fragen

1.Was beschreibt das Interface AssocStringString und wozu kann es verwendet werden?

Interface legt Struktur fest und dabei wird ein string als Schlüssel festgelegt und schließlich als Wert einen string ausgegeben. Er kann dazu verwendet werden, um über einen Schlüssel einfach auf das gewünschte Objekt zugreifen zu können. Meist um homogenes assoziatives Array (AssocStringString).

2.Was geschieht in Zeile 9 und was ist das Ergebnis?

Variable „Port“ wird festgelegt. Dieser ist dafür da, damit Sever weiss auf welchen Port er achten muss. Man setzt ihn auf eine Zahl, damit er nicht undefiniert ist und der Browser auf die Applikation zugreifen kann

3.Was geschieht in Zeile 25 von ServerTest.js?

let query: AssocStringString = Url.parse(_request.url, true).query

in Datentyp AssocStringString wird neue Variable definiert. Dieser wird Url.parse(_request.url, true).query zugewiesen. Dabei wird ein Url string in url Objekt umgewandelt durch url.parse. query wird ein neues Objekt von AssocStringString, wenn url.parse richtig ist. Und durch query wird dann auf den query Teil der url zugegriffen

4.Wie arbeitet die for-in Schleife in Zeile 29?

For-in-Schleife durchläuft das Objekt bzw. die Eigenschaften der Variablen query. Nicht beachtet werden Teile des Arrays, welche nicht den Schlüssel key enthalten.

5.Was bewirkt die Header-Information Acces-Control-Allow-Origin?

Ist ein Mechanismus der dem Browser ein Cross.Origin-Request ermöglicht und eine Kommunikation des Browsers erlaubt.
